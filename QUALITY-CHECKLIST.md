# FREAKMATH 시각화 품질 체크리스트

모든 시각화 작업 시 반드시 확인할 것:

## 1. 겹침 방지
- [ ] 모든 텍스트 라벨 간 y좌표 최소 0.8 간격
- [ ] 애니메이션 단계별로 텍스트가 동시에 2개 이상 같은 영역에 안 나옴
- [ ] 단계 전환 시 이전 단계 텍스트는 사라짐
- [ ] 수식과 설명 텍스트는 항상 다른 영역에 배치

## 2. 슬라이더 (직접 해보기)
- [ ] 모든 값은 v('변수명', 기본값) 사용 (하드코딩 금지)
- [ ] 소수점 지원: parseFloat(toFixed(1)) 사용, Math.round 금지
- [ ] 슬라이더 값 변경 시 Canvas에 즉시 반영
- [ ] 슬라이더 섹션은 애니메이션 아님 — 현재 값의 정적 상태
- [ ] fallback 원+텍스트 절대 사용 금지

## 3. 텍스트
- [ ] 모든 텍스트 한국어 (영어 금지)
- [ ] 특수 수학 기호(∫, Σ, √) 대신 한국어 설명 사용
- [ ] 폰트 크기: gText 최소 13px 이상
- [ ] 글로우 텍스트(gText) 사용으로 가독성 확보

## 4. 캔버스
- [ ] 시각화 캔버스 높이 최소 600px
- [ ] 슬라이더 캔버스 높이 최소 500px
- [ ] 모든 요소가 캔버스 안에 들어감 (잘림 없음)
- [ ] W, H 기반 동적 스케일 (Math.min으로 오버플로 방지)

## 5. 애니메이션
- [ ] 총 8~15초, 끝나면 3초 대기 후 반복
- [ ] p 기반 단계 분기 (0~1)
- [ ] easeOut/easeInOut 이징 적용
- [ ] 단계별 텍스트 (①②③...) 포함

## 6. 색상
- [ ] 보라(#534AB7), 초록(#1D9E75), 주황(#D85A30) 3색 체계
- [ ] 다크모드 대응
- [ ] gLine, gCircle, gText 헬퍼 함수 사용

## 7. 수학 정확성
- [ ] 공식이 수학적으로 올바른지 확인
- [ ] 계산 결과에 Number.isFinite 체크
- [ ] NaN/undefined 방어 코드
